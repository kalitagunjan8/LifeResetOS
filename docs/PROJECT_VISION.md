# PROJECT VISION

Project: Life Reset OS

Version: Living Document

---

# Mission

Life Reset OS is a personal operating system designed to help people transform their lives through intentional action.

It is not a habit tracker.

It is not a to-do list.

It is not a productivity app.

Its purpose is to convert a person's life mission into daily execution.

---

# Core Philosophy

Everything begins with one question:

"What kind of life do I want to build?"

The application helps the user answer that question and then breaks it into executable actions.

The hierarchy is fixed:

Mission
↓
Goals
↓
Tasks (Daily Actions)
↓
Focus Sessions

Every feature must strengthen this hierarchy.

Nothing should bypass it.

---

# Product Principles

## 1. Mission First

Every user should always know what their current Mission is.

The Mission is the highest priority object in the application.

---

## 2. Goals Create Direction

Goals exist only to achieve the Mission.

Goals should never become isolated projects.

Every Goal must belong to one Mission.

---

## 3. Tasks Create Progress

Tasks are the smallest meaningful unit of work.

Completing Tasks creates Goal progress.

Goal progress creates Mission progress.

---

## 4. Focus Creates Momentum

Users do not focus on Goals.

Users focus on Tasks.

Focus Sessions should therefore attach to Tasks.

---

## 5. Progress Must Be Earned

Progress should be calculated from completed work.

The application should avoid fake progress indicators.

Progress should never be manually adjustable.

---

# What Life Reset OS Is NOT

The project intentionally avoids becoming:

- Generic habit tracker
- Generic to-do list
- Calendar replacement
- Note-taking application
- Project management software
- Social media platform

If a feature does not strengthen the Mission → Goal → Task hierarchy, it should not be added.

---

# Design Philosophy

The interface should feel:

- Calm
- Minimal
- Intentional
- Focused
- Encouraging

Avoid unnecessary animations, clutter, or visual noise.

The product should reduce cognitive load rather than increase it.

---

# Development Philosophy

Build foundations before features.

Prefer simple architecture over clever architecture.

Avoid premature optimization.

Every new feature should integrate cleanly into the existing hierarchy.

When in doubt:

Choose maintainability over speed.

---

# AI Collaboration Principles

AI assistants should:

- Respect previous architectural decisions.
- Avoid introducing unnecessary complexity.
- Prefer incremental development.
- Keep commits small and stable.
- Update project documentation after meaningful milestones.

AI should not redesign previously accepted architecture without strong justification.

---

# Long-Term Roadmap

Current architecture:

Mission
↓
Goals
↓
Tasks
↓
Focus Sessions

Future layers include:

- Daily Review
- Weekly Review
- Analytics
- Life Areas
- Long-term Insights

These should extend the existing hierarchy rather than replace it.

---

# Definition of Success

A successful Life Reset OS user should:

Know their Mission.

Know today's important Tasks.

Complete focused work consistently.

See measurable progress toward meaningful goals.

The application succeeds when it helps users build a better life through consistent execution.
